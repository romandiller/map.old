terramark - поиск земельных участков

	ssl - сертификат

	Перед отправкой в релиз, необходимо собрать фронтенд. Для этого загляните в package.json. Нужно выполнить gulp & webpack

		gulp для сборки css
		webpack для сборки frontend javascript

	кейсы:

		- поиск участка по клику в точку (точечный поиск)
		- массовый поиск (выделение участка на карте, дальше фронт делает множественные запросы в росреестр)
		- выбор региона из списка и смещение карты на точку

	Глобальный поиск:

		
		строим вделенную область (прямоугольник)
		показываем кнопку глобального поиска
		создаем массив с координатами
		в цикле отправляем координаты на сервер и показываем результат

backend

	routes & actions:

		*** каждый route = отдельный folder

		routes - get запросы - отображение страниц

			/ - главная - описание сервиса
			/kadastrovaya-karta - кадастровая карта
			/articles - статьи
			/articles/:url - конкретная статья
			/sitemap - карта сайта

		actions - socket.io & fetch запросы

			/gridTilesGrid - запрос для тайлов кадастровой карты
			/regionsList - получить списка регионов из базы
			/getRegion - получить регион по координатам
			/searchParcel - поиск участка по выбранным координатам
			/searchParcelList - поиск нескольких участков по выбранному объекту			

	Модули

		main - точка входа
		eventEmitter - регистрация и вызов методов
		ejs - шаблонизотор
		logger - логирование
		session - сессии клиента, если нужно
			cookie
		mailer - отправка email
		request (superagent) - запросы в pkk5 и Росреестр
		router - маршрутизатор

			/folder/route - маршрут с данными
			/api/:route - fetch запросы

		ws_router - маршрутизатор ws запросов
		db - mongodb & mysql
		api - класс для загрузки данных от росреестра
			getGrid - построение сетки 
			getParcell - поиск участка
